# 상담신청 → 구글 시트 연동 (SheetDB 방식)

상담 폼 신청이 들어오면 **SheetDB**를 통해 마스터 구글 시트의
**`김훈찬변호사님랜딩` 탭**에 한 줄씩 자동으로 쌓입니다.

## 현재 설정 (작동 확인됨)
- 커넥터: **SheetDB** (`https://sheetdb.io/api/v1/13qwziu514dyb`)
- 대상 시트: 사무실 마스터 시트 파일 (`1TLNvo6...`)
- 대상 탭: **`김훈찬변호사님랜딩`** (보호/수식 충돌을 피하려고 별도 전용 탭 사용)
- 탭 1행 헤더 (정확히 일치해야 함):
  ```
  날짜 | 시간 | 요일 | 1차 유입경로 | 성명 | 연락처 | 상담항목 | 특이사항
  ```

## 들어가는 값
| 열 | 값 |
|---|---|
| 날짜 / 시간 / 요일 | 신청 시각 (한국시간 자동) |
| 1차 유입경로 | `홈페이지, 전화` (통합시트와 동일 분류 → 나중에 합치기 쉬움) |
| 성명 / 연락처 / 상담항목 | 폼 입력값 |
| 특이사항 | `김훈찬변호사님 렌딩` (고정) |

## 코드/환경변수
- 처리 코드: [src/app/api/consult/route.ts](src/app/api/consult/route.ts)
- 환경변수 (`.env.local`, 그리고 **Vercel에도 동일하게 등록 필요**):
  ```
  SHEET_API_URL=https://sheetdb.io/api/v1/13qwziu514dyb
  SHEET_TAB=김훈찬변호사님랜딩
  ```

## 주의 / 참고
- **탭 1행 헤더를 지우거나 이름을 바꾸면 기록이 깨집니다.** (특히 `1차 유입경로`, `특이사항` 글자 그대로 유지)
- 마스터 시트의 `통합시트`/`홈페이지, 전화` 등 다른 탭은 **보호되어 있거나 수식 뷰**라 직접 기록 불가 → 그래서 전용 탭(`김훈찬변호사님랜딩`)을 씁니다.
- 광고/키워드 분석은 시트가 아니라 **`/admin` 대시보드**에서 봅니다.
  연락처 등 개인정보는 시트에만, 행동분석 DB(Supabase)에는 이름·전화번호를 저장하지 않습니다.
- SheetDB 무료 플랜은 월 요청 수 한도가 있습니다. 신청량이 많아지면 플랜 확인 필요.
